Code example

<label for="fullname">What is your name?</label>
<input id="name" name="name" type="text" value="" />

A form field is preceded by a label asking for a name but the label is not correctly linked to the question due to a typo on the for attribute. The results would be the same if there was no for attribute.

Why it’s a problem

It is a problem for the same reasons of not having a label at all.

How to test for it

This can be tested the same way as not having a label at all.

How to fix it

The for attribute of the label must always match the id attribute of the form field.

<label for="name">What is your name?</label>
